# Godot Intro Sequence — Data-Driven Screen Sequencer

A minimal Godot 4 demo showing how to replace an if/else intro chain with a data array. The entire sequence — logos, text screens, localized strings — is driven by a single `_SCREENS` const. Adding or reordering screens requires no code changes.

Extracted from [**Dopros**](https://store.steampowered.com/app/4348260/DOPROS/) by [The Witches Circle](https://thewitchescircle.games/).

---

## What it demonstrates

### 1. Array-as-config sequencer (`intro_sequence.gd`)

```gdscript
const _SCREENS: Array[Dictionary] = [
	{ "type": &"logo", "path": "res://assets/logos/studio.png",  "duration": 4.0 },
	{ "type": &"logo", "path": "res://assets/logos/engine.png",  "duration": 4.0 },
	{ "type": &"text", "text": "RAW STRING — no translation.",   "duration": 7.0 },
	{ "type": &"text", "key": "INTRO_ATTRIBUTION",               "duration": 7.0 },
]
```

A single `_show_screen()` method handles both types, the fade tween, and the auto-advance timer. ~70 lines total.

### 2. Two ways to put text on a screen

| Field | Behaviour |
|-------|-----------|
| `"text"` | Displayed as-is — useful for legalese you don't want translated |
| `"key"` | Passed through `tr()` — pulls from TranslationServer |

The distinction is explicit in the data, not buried in branching code.

### 3. Input handling

- `ui_accept` (Enter / Space / **gamepad South**) or mouse click → advance one screen - `ui_cancel` (Escape / **gamepad East**) → skip the whole sequence immediately

Both actions use Godot's built-in input map, which already includes the standard gamepad buttons — no extra input setup needed. The end screen button grabs focus automatically on `_ready()`, so a controller player can quit without touching the keyboard.

## Getting started

1. Clone or download this repository.
2. Open the project folder in **Godot 4.4+**.
   Godot will auto-import assets and compile `locale/ui.csv` into `.translation` binaries on first open.
3. Press **F5** to run.

## Adding your own screens

Open `scenes/intro_sequence/intro_sequence.gd` and edit `_SCREENS`:

```gdscript
# Another logo
{ "type": &"logo", "path": "res://assets/logos/my_logo.png", "duration": 3.0 },

# Plain text (no localization)
{ "type": &"text", "text": "MADE WITH LOVE IN SOMEWHERE.", "duration": 5.0 },

# Localized text — add the key to locale/ui.csv
{ "type": &"text", "key": "MY_WARNING_KEY", "duration": 8.0 },

# Localized text with an optional small icon above it
{ "type": &"text", "key": "MY_WARNING_KEY", "icon": "res://assets/my_icon.png", "duration": 8.0 },
```

The `"icon"` field is optional on any `"text"` screen. Omit it (or leave it as `""`) and no image is shown — the text is simply centered on its own.

The `"duration"` field is optional on every screen type — omit it to fall back to the **5 s default**.

To add a translation: edit `locale/ui.csv` and add a column for the new locale, then re-import in Godot editor and register the new `.translation` file in Project Settings → Internationalization → Translations.

---

## Localization note

`locale/ui.csv` is the source file — human-readable, version-controlled. The compiled `*.translation` binaries are generated by Godot on import and are excluded from the repository (see `.gitignore`). Open the project in Godot once to generate them before running.
